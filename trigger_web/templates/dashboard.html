How It Works

Time triggers monitor DNS queries from specified device groups. When a trigger domain is accessed, a timer starts. After the time limit expires, blocking is applied using one of two modes:

  • Regex: Adds a regex deny rule to block matching domains
  • Adlist: Associates a Pi-hole blocklist with the trigger's groups

Daily Reset

All triggers automatically reset at {{ '%02d:00'|format(reset_hour) }} local time. This removes all active blocks and clears all timers, giving a fresh start each day.
